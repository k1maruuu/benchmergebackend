== Введение

Этот документ содержит описание начальной конфигурации проекта BenchMerge Backend, включая требования к зависимостям и конфигурацию Docker-контейнера.

== Зависимости проекта

Файл `requirements.txt` содержит список всех Python-библиотек, необходимых для работы проекта.

== Docker конфигурация

Файл `Dockerfile` описывает процесс сборки контейнера для развертывания проекта.

=== Полный код Dockerfile

[source,dockerfile]
----
FROM python:3.12-slim

WORKDIR /app

ENV PYTHONDONTWRITEBYTECODE=1 \
    PYTHONUNBUFFERED=1

COPY requirements.txt ./
RUN pip install --no-cache-dir -r requirements.txt

COPY app ./app

EXPOSE 8100

CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8100"]
----

=== Описание этапов сборки

|===
| Инструкция | Описание

| `FROM python:3.12-slim`
| Использует официальный образ Python 3.12 в минималистичной версии

| `WORKDIR /app`
| Устанавливает рабочую директорию внутри контейнера

| `ENV PYTHONDONTWRITEBYTECODE=1 PYTHONUNBUFFERED=1`
| Отключает кэширование байт-кода и буферизацию вывода Python для корректной работы логов

| `COPY requirements.txt ./`
| Копирует файл зависимостей в контейнер

| `RUN pip install --no-cache-dir -r requirements.txt`
| Устанавливает все зависимости без кэширования (уменьшает размер образа)

| `COPY app ./app`
| Копирует исходный код приложения

| `EXPOSE 8100`
| Объявляет порт 8100 как используемый контейнером

| `CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8100"]`
| Запускает uvicorn сервер, который слушает все интерфейсы на порту 8100
|===

=== Запуск контейнера

Для запуска контейнера используйте следующую команду:

[source,bash]
----
docker build -t benchmerge-backend .
docker run -p 8100:8100 benchmerge-backend
----

=== Переменные окружения

Dockerfile устанавливает следующие переменные окружения:

* `PYTHONDONTWRITEBYTECODE=1` - предотвращает создание файлов `.pyc`
* `PYTHONUNBUFFERED=1` - отключает буферизацию stdout/stderr для немедленного вывода логов

== Архитектура запуска

После запуска контейнера приложение будет доступно по адресу `http://localhost:8100`.

== Заключение

Проект BenchMerge Backend использует современные технологии:

* FastAPI для создания API
* Pydantic для валидации данных
* Уvicorn как ASGI сервер
* Docker для контейнеризации

Минимальные версии зависимостей гарантируют совместимость и стабильность работы приложения.
